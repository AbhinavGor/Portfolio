SafePass.com

Learn how PHP can generate safe & secure passwords.

SafePass.com is a website with a PHP backend which generates random secure passwords according to your requirements.

Here is a brief explanation of all the functions I used to build this website and how they work. The website allows users to generate secure passwords. I started up with setting up checkboxes to allow users to select what all sets of characters they want in their passwords. The character set options include:

if(isset($_POST['Uletters']))
{
	$charset = $charset.'ABCDEFGHIJKLMNOPQRSTUVWXYZ';
}
if(issert($_POST['Lletters']))
{
	$charset = $charset.'abcdefghijklmnopqrstuvwxyz'
}
if(isset($_POST['Numbers']))
{
	$charset = $charset.'1234567890';
}
if(isset($_POST['Special']))
{
	$charset = $charset.'%$&@!\'\"\\/.,[](){}^~`'
}

Users get to select the character select the character sets which get appended to the initial empty set of characters. This set of characters then get passed on to a random string generator. This "random string generator" I'm mentioning here is basically a string shuffler function pre-defined in PHP. The character set that we pass onto the random string generator is nothing but a very long string of integers, characters and special characters. The exact syntax of the code needed to shuffle the string is "str_shuffle($charset)". This set of characters gets shuffled by the inbuilt PHP function and then sent on for further processing. This "further processing" involves MD5 Hashing.

I'm not going to discuss more on MD 5 Hashing as that would compromise the integrity of the website. The form handling method used here is POST method. This ensures that sensitive details aren't given away from the URL. MD5 hashing is also not applied for all the combinations of character sets provided by the user.

The user gets a password according to their needs after submitting the form. This is a screenshot of the final hosted website which you can find here.

My future plans for this project include using better hashing algorithms like SHA 1 or using an actual crytography agorithm instead of a message-digest algorithm like MD5 or SHA.
